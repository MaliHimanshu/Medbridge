# Contributing to MedBridge

Thank you for your interest in contributing to MedBridge!

We welcome contributions that improve the project, fix bugs, or add new features.

---

## Getting Started

1. Fork the repository.

2. Clone your fork.

```bash
git clone https://github.com/MaliHimanshu/Medbridge.git
```

3. Install dependencies.

Backend

```bash
cd backend
npm install
```

Frontend

```bash
cd frontend
npm install
```

---

## Create a Branch

```bash
git checkout -b feature/your-feature-name
```

---

## Coding Guidelines

- Write clean code.
- Use meaningful variable names.
- Follow existing project structure.
- Test before committing.
- Never commit `.env` files.
- Never commit `node_modules`.

---

## Commit Changes

```bash
git add .
git commit -m "Add new feature"
git push origin feature/your-feature-name
```

---

## Create Pull Request

Open a Pull Request and describe:

- What you changed
- Why you changed it
- Screenshots (if UI changes)

---

## Reporting Issues

Please include:

- Operating System
- Node.js Version
- Steps to Reproduce
- Expected Result
- Actual Result
